rust-norion is an open-source Rust AI control-layer prototype focused on routing, memory, reflection, runtime boundaries, evidence gates, rollback, and auditable self-evolution.

It is not a production LLM inference kernel or a wrapper around one model API. The project explores how Rust can make the layer around inference more explicit, testable, local-first, and reviewable.

Areas where contributors can help:

- Rust control-layer architecture, routing, reflection, scheduler, and writer gates
- memory systems, KV/gist memory, retrieval hygiene, and experience replay
- runtime adapter traits, manifests, command runtimes, and device profiles
- benchmark, CI, trace schema, and reproducible gates
- docs, runbooks, diagrams, and contributor onboarding
- clean-room review, license boundaries, privacy, and governance

Feedback and focused PRs are welcome.
